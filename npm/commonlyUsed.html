# 举个栗子

# or
npm home lodash

3

npm docs或者npm home命令在不接参数时，会在当前项目中，通过 package.json 文件中的homepage配置，来打开对应的地址。

其原理就是：当你要查看的项目中的 package.json 文件中，设置了homepage属性，通过npm docs/home就能打开对应的主页，没有设置homepage属性时，npm 会继续寻找其中的repository属性，这时候打开的就是项目在 github 中的托管地址 url npm 官网中包的所在地址，（例如：www.npmjs.com/package/npm…open in new window

# npm 发布包

注意:使用淘宝源会报错，要改回默认源

# 修改下版本号

npm version [版本号]
1

# 当前目录执行npm publish

npm publish
1

# 弃用包的相关操作

注意：强烈建议弃用包或包版本而不是取消发布它们，因为取消发布会从注册表中完全删除一个包，这意味着任何依赖它的人都将无法再使用它，而不会发出警告。

# 弃用整个包

npm deprecate package-name "弃用信息"
1

# 弃用包的单个版本

npm deprecate package-name@version "弃用信息"
1

# 取消弃用操作

# 将弃用消息改为空字符串即可
npm deprecate package-name ""
1
2

# 取消发布整个包

npm unpublish [package-name] -f
1

# 取消发布包的指定版本

npm unpublish [package-name]@<version>
1
