# XMLHttpRequest

XMLHttpRequest（XHR）对象用于与服务器交互。通过 XMLHttpRequest 可以在不刷新页面的情况下请求特定 URL，获取数据。这允许网页在不影响用户操作的情况下，更新页面的局部内容。XMLHttpRequest 在 AJAX 编程中被大量使用。

尽管名称如此，XMLHttpRequest 可以用于获取任何类型的数据，而不仅仅是 XML。它甚至支持 HTTP 以外的协议（包括 file:// 和 FTP），尽管可能受到更多出于安全等原因的限制。

如果您的通信流程需要从服务器端接收事件或消息数据，请考虑通过 EventSourceopen in new window 接口使用 server-sent eventsopen in new window。对于全双工的通信， WebSocketopen in new window 可能是更好的选择。

# 构造函数

XMLHttpRequest()

该构造函数用于初始化一个 XMLHttpRequest 实例对象。在调用下列任何其他方法之前，必须先调用该构造函数，或通过其他方式，得到一个实例对象。

# 属性

此接口继承了 XMLHttpRequestEventTargetopen in new window 和 EventTargetopen in new window 的属性。

XMLHttpRequest.onreadystatechangeopen in new window

当 readyState 属性发生变化时，调用的 EventHandleropen in new window (en-US)。

XMLHttpRequest.readyStateopen in new window 只读

返回 一个无符号短整型（unsigned short）数字，代表请求的状态码。

XMLHttpRequest.responseopen in new window 只读

返回一个 ArrayBufferopen in new window、Blobopen in new window、Documentopen in new window，或 DOMStringopen in new window，具体是哪种类型取决于 XMLHttpRequest.responseTypeopen in new window 的值。其中包含整个响应实体（response entity body）。

XMLHttpRequest.responseTextopen in new window 只读

返回一个 DOMStringopen in new window，该 DOMStringopen in new window 包含对请求的响应，如果请求未成功或尚未发送，则返回 null。

XMLHttpRequest.responseTypeopen in new window

一个用于定义响应类型的枚举值（enumerated value）。

XMLHttpRequest.responseURLopen in new window 只读

返回经过序列化（serialized）的响应 URL，如果该 URL 为空，则返回空字符串。

XMLHttpRequest.responseXMLopen in new window 只读

返回一个 Document，其中包含该请求的响应，如果请求未成功、尚未发送或时不能被解析为 XML 或 HTML，则返回 null。

XMLHttpRequest.statusopen in new window 只读

返回一个无符号短整型（unsigned short）数字，代表请求的响应状态。

XMLHttpRequest.statusTextopen in new window 只读

返回一个 DOMStringopen in new window，其中包含 HTTP 服务器返回的响应状态。与 XMLHTTPRequest.statusopen in new window 不同的是，它包含完整的响应状态文本（例如，"200 OK"）。

**注意：**根据 HTTP/2 规范（8.1.2.4 Response Pseudo-Header Fieldsopen in new window，响应伪标头字段），HTTP/2 没有定义任何用于携带 HTTP/1.1 状态行中包含的版本（version）或者原因短语（reason phrase）的方法。

XMLHttpRequest.timeoutopen in new window

一个无符号长整型（unsigned long）数字，表示该请求的最大请求时间（毫秒），若超出该时间，请求会自动终止。

XMLHttpRequestEventTarget.ontimeout

当请求超时调用的 EventHandleropen in new window (en-US)。

XMLHttpRequest.uploadopen in new window 只读

XMLHttpRequestUpload，代表上传进度。

XMLHttpRequest.withCredentialsopen in new window

一个布尔值 (en-US)open in new window，用来指定跨域 Access-Control 请求是否应当带有授权信息，如 cookie 或授权 header 头。

# 非标准属性

XMLHttpRequest.channelopen in new window 只读

一个 nsIChannel，对象在执行请求时使用的通道。

XMLHttpRequest.mozAnonopen in new window 只读

一个布尔值，如果为真，请求将在没有 cookie 和身份验证 header 头的情况下发送。

XMLHttpRequest.mozSystemopen in new window 只读

一个布尔值，如果为真，则在请求时不会强制执行同源策略。

XMLHttpRequest.mozBackgroundRequestopen in new window

一个布尔值，它指示对象是否是后台服务器端的请求。

# 事件处理器

作为 XMLHttpRequest 实例的属性之一，所有浏览器都支持 onreadystatechange。

后来，许多浏览器实现了一些额外的事件（onload、onerror、onprogress 等）。详见Using XMLHttpRequestopen in new window。

更多现代浏览器，包括 Firefox，除了可以设置 on* 属性外，也提供标准的监听器 addEventListener()open in new window API 来监听XMLHttpRequest 事件。

# 方法

XMLHttpRequest.abort()open in new window

如果请求已被发出，则立刻中止请求。

XMLHttpRequest.getAllResponseHeaders()open in new window

以字符串的形式返回所有用 CRLFopen in new window 分隔的响应头，如果没有收到响应，则返回 null。

XMLHttpRequest.getResponseHeader()open in new window

返回包含指定响应头的字符串，如果响应尚未收到或响应中不存在该报头，则返回 null。

XMLHttpRequest.open()open in new window

初始化一个请求。该方法只能在 JavaScript 代码中使用，若要在 native code 中初始化请求，请使用 openRequest()open in new window。

XMLHttpRequest.overrideMimeType()open in new window

覆写由服务器返回的 MIME 类型。

XMLHttpRequest.send()open in new window

发送请求。如果请求是异步的（默认），那么该方法将在请求发送后立即返回。

XMLHttpRequest.setRequestHeader()open in new window

设置 HTTP 请求头的值。必须在 open() 之后、send() 之前调用 setRequestHeader() 方法。

# 非标准方法

XMLHttpRequest.init()

在 C++ 代码中初始化一个 XHR 对象。

**警告：**该方法不能在 JavaScript 代码中使用。

XMLHttpRequest.openRequest()open in new window

初始化一个请求。这个方法只能在原生 C++ 代码中使用；如果用 JavaScript 代码来初始化请求，使用 open()open in new window 代替。可参考 open() 的文档。

XMLHttpRequest.sendAsBinary()

send() 方法的变体，用来发送二进制数据。

# 事件

也可以使用 onabortopen in new window 属性。

erroropen in new window

当 request 遭遇错误时触发。

也可以使用 onerroropen in new window 属性

loadopen in new window

XMLHttpRequestopen in new window请求成功完成时触发。

也可以使用 onloadopen in new window 属性.

loadendopen in new window

当请求结束时触发, 无论请求成功 (loadopen in new window) 还是失败 (abortopen in new window 或 erroropen in new window)。

也可以使用 onloadend 属性。

loadstartopen in new window

接收到响应数据时触发。

也可以使用 onloadstartopen in new window 属性。

progressopen in new window

当请求接收到更多数据时，周期性地触发。

也可以使用 onprogressopen in new window 属性。

也可以使用 ontimeout 属性。
